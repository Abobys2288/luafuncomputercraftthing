# NFPC — Compressed NFP Format

**NFPC** (NFP Compressed) — это текстовый формат изображений для CCOS, совместимый с палитрами NFP (32 цвета) и NFP256 (256 цветов), но с построчным RLE-сжатием для экономии места.

## Заголовок

Первая строка файла:
```
!NFPC <width> <height> <mode>
```
- `mode` — `32` или `256`.

Пример:
```
!NFPC 64 64 256
```

## Кодирование строки

Каждая последующая строка — это одна строка пикселей, закодированная побеговым методом (RLE).

- **Одиночный пиксель** записывается как есть:
  - 32-color: один символ из `0-9a-v`
  - 256-color: два hex-символа, например `01` или `fc`

- **Повторяющаяся серия** (count ≥ 2) записывается как:
  ```
  ~<pixel><count_hex>
  ```
  - `~` — маркер RLE.
  - `<pixel>` — 1 символ (32) или 2 hex (256).
  - `<count_hex>` — **ровно 2 hex-цифры** (нижний регистр), от `02` до `ff`. Максимум 255 пикселей в одном блоке. Если серия длиннее — она разбивается на несколько блоков.

## Пример

32-color, строка из 64 белых пикселей (`1`):
```
~140
```
(64₁₀ = 40₁₆)

256-color, строка из 64 белых пикселей (`01`):
```
~0140
```

## Экономия

| Файл | Исходный | NFPC | Экономия |
|------|----------|------|----------|
| `smile.nfp` (64×64, 32) | 4159 B | 1562 B | **62 %** |
| `smile.nfp256` (64×64, 256) | 8255 B | 2429 B | **71 %** |

На иконках и UI-графике с большими одноцветными областями выигрыш максимальный. На фотографиях с мелким шумом выигрыш меньше, но размер никогда не превышает оригинал.

## C2 — улучшенная ветка сжатия

Новая ветка остаётся в расширениях `.nfpc` и `.nfpa`, но добавляет маркер `C2` в заголовок:
```
!NFPC <width> <height> <mode> C2
!NFPA <width> <height> <mode> <delay_ms> <loop> <frames> C2
```

Старые заголовки без `C2` продолжают читаться как обычный RLE.

В C2 каждая строка выбирается конвертером как самый короткий из вариантов:
- обычная legacy RLE-строка;
- `!<base64>` — packed raw row (`mode 256`: 1 байт на пиксель, `mode 32`: два 4-bit пикселя в байте);
- `=` — повтор предыдущей строки текущего кадра/картинки;
- `^` — повтор строки с тем же `y` из предыдущего кадра (`.nfpa` only).

Для анимаций это особенно важно: неизменные строки между кадрами занимают один символ, а noisy 256-color строки больше не раздуваются до двух hex-символов на пиксель.

## C3 — потоковое сжатие для больших кадров

Для больших `.nfpa` web-конвертер дополнительно пробует `C3` и выбирает самый маленький результат среди legacy RLE, C2 и C3.

Заголовки:
```
!NFPC <width> <height> <mode> C3
!NFPA <width> <height> <mode> <delay_ms> <loop> <frames> C3
```

Тело C3 хранится chunked base64-блоками:
```
@<chunk_count>
<base64 chunk 1>
<base64 chunk 2>
...
```

Внутри base64 лежит bytecode по пикселям в raster order. Верхние 2 бита команды — тип, нижние 6 бит — длина минус 1 (`1..64`):
- `00` — copy from previous frame at the same pixel offset (`.nfpa`);
- `01` — copy from the previous row of the current frame;
- `10` — solid run, next byte is palette index;
- `11` — raw run, next `len` bytes are palette indices.

C3 особенно выигрывает на GIF/video, где много областей не меняется между кадрами. Для статичных картинок с простыми областями C2 может оказаться меньше, поэтому конвертер выбирает автоматически.

## Инструменты

- `tools/convert_to_nfp.py` — конвертация PNG → `.nfpc` (`--nfpc` или расширение `.nfpc`).
- `tools/convert_nfp_to_nfpc.py` — конвертация существующих `.nfp`/`.nfp256` → `.nfpc`.
- `tools/convert_to_nfp.html` — standalone web-конвертер с legacy RLE и C2 для `.nfpc`/`.nfpa`.

## Поддержка в CCOS

- **Image Viewer** (`imgview`) — открывает `.nfpc` (32 и 256).
- **Boot logo** (`init.lua`) — загружает `bootlogo.nfpc`.
- **Desktop** (`desktop.lua`) — иконки программ в `.nfpc`.
- **File Manager** — распознаёт `.nfpc` как изображения.
